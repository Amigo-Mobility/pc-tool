CSLLC PC Tool

CSLLC PC Tool combines multiple GUI Software packages on a single launch platform. The Software packages include:

  • CS1108 Motor Controller Programmer
  • CS1520 Programmer

Release Package:

The software package includes a zip file containing:

  • bmp_files directory
  • html_files directory
  • Images directory
  • print_files directory
  • com_port.ini file
  • installed_peripherals.json file
  • cs1520_programmer executable
  • cs1108_programmer executable

bmp Files Directory

The bmp files directory contains the images loaded as illustrations in the operator manual.

html_files directory

The html files directory contains htm/html files loaded at programmer run-time. HTM/HTML files include:

  • html_header.htm : CS1520 programmer header section
  • html_file_end.htm : CS1520 programmer footer section
  • markers_length_lat_lng.htm : HTM file invoked when plotting points
  • operator_manual.htm : Programmer operator manual

Images Directory

The Images directory contains Images loaded at programmer run-time conditional on user response.

Print Files Directory

The Print_files directory contains the label and excel files used in printing the CS1520 Programmer Plot Label. The files include:

  • cas_label.csv : comma separated value (excel) file
  • cas_label.lbx : Brothers printer label file

com_port.ini file

This file contains the port assignment for the USB Dongle and for the USB Connection to the CS1520 Programmer card.

installed_peripherals.json File

The installed peripherals.json (Java Script Object Notation) file controls the peripheral options that are activated at run-time.

session_data.ini : Session data Repository

The Session Data ini file captures and stores the directory where the Profile Files are stored, the directory where the Label Files are stored, the directory where the log file is stored, and captures the Profile Records added to the system during the session. Session Data Row By Row Definition:

  • 1) Profile (*.dat) Files Directory
  • 2) Label (*.lbx) Files Directory
  • 3) Log (*.log) File Name including Directory Path
  • 4) Bitwise variable identifying implemented peripherals
  • 5 to End) List of Profiles Entered into System

Installation Procedure:

Download the zp1 file attachment and rename file to type *.zip

When the download is complete double click on the *.zip file to unzip. Click on the zip file to start extracting files: File->Extract All...Extraction Wizard will appear

Setup Communication Port Assignment

There are two ports that need to be setup to enable communication with programmable devices:

  • CS1108 USB Dongle Port
  • CS1520 Programming Port

USB Dongle Setup

In order to program coordinate points for Halt and Warning Boundary areas, the PC needs to connect to the target over USB.

Connect the CS1520 Programming Cable to one of the PC's USB connections.

The communication port mapped to the device can be observed via the Control Panel: Start->Control Panel->Hardware(tab)->Device Manager. Click on the Device Manager tab to display devices and expand the Ports (COM & LPT) to display the USB Serial Port.

When the selected communication port matches the Device Manager Mapped Port, the device is ready to be programmed.

Port Assignments

Take note of the COM Port number assigned to the USB Ports as follows:

  • USB Dongle : Silicon Labs CP210x USB to UART Bridge
  • CS1520 Programmer : CS1520 Serial Port

CS1520 Programmer Options

The CS1520 Programmer provides a method for selecting a pre-defined file of latitude/longitude coordinate pairs representing Halt and Warning Boundary areas of operation. Operation of a CS1520 populated mobile vehicle is impacted as follows:

  • Vehicle Outside Halt Boundary - Vehicle Inoperable
  • Vehicle Outside Warning Boundary - Warning Indication
  • Vehicle Inside Warning Boundary - Unihibited Operation
    • HELP : Access Help Screen to Further describe options
    • LOAD FILE : Load an existing Halt/Warning Boundary Areas File Load Plot File
    • PROG UUT : Write Hal/Warning Boundary Areas to Device Program UUT
    • EXIT : Exit the Session

    LOAD FILE

    The Load File option loads a file of plot points to be programmed into the device. The option initiates an "OPEN FILE" dialog as show below. The User will navigate the browser to the location of the plot file to be loaded and in turn written to the device.

    PROG UUT

    The PROG UUT option writes the Warning and Halt Boundary areas to the mobile device.Once the User navigates to the location of the plot file, selects the file, and clicks on the "OPEN" button, control returns to the main screen where the name of the file loaded is identified.

    A time critical sequence exists when attempting to program the target. On power up the device "wakes up" for a short period of time then returns to "sleep". This short "awake" time is insufficient for programming the device. To solve this issue, a suspend command is initiated immediately after power cycle. In addition, a button press of the red button, as depicted in the following image, needs to occur immediately after power on, after which the "PROG UUT" button is clicked to commence programming.

PC TOOL HELP

The PC Tool Help file provides a reference between the options available on the display screens and the function performed by those options. This Help display is accessed by clicking the "HELP!" button from any of the PC Tool display pages. The Help file includes some hierachy discussion and a narrative on each of the individual button options.

CONNECT TO TARGET

The connect button allows the User to change the CS Controller that is connected to the tool. The User would disconnect the current Motor Controller and remove from the bench. A new Motor Controller is then placed next to the system with the EEPROM, control and power connections completed. Once the new Motor Controller is connected the User will press the Connect button to query the Motor Controller for the Device ID, Serial Number, and Firmware Revision. The information is displayed on an acknowledge banner where the User acknowledges the changed Motor Controller by clicking the OK button. Use of the PC Tool resumes as before.

SETTINGS

Settings are password protected options. When the "Settings" button is selected, the User will be prompted to enter the password.

ENTER PASSWORD

Enter the password in the "white" textbox to the right of the "ENTER PASSWORD" caption . If the password is valid, control advances to the Settings Panel, otherwise control returns to the previous Panel.

ENTER

Press the "ENTER" button when the password entry is complete. If the password entered is valid, control advances to the Settings Panel.

Settings Panel Options:

Set Comm Port

Adjacent to the "COMM PORT" field is a selection box containing a list of possible Communication Ports where the USB Dongle may have been connected. By selecting one of these comm ports from the list, the User initializes the USB Dongle Port Setting to the port where the Dongle is connected.

"Create"/"Browse" Log File

The Log file is the repository for capturing and recording Programming Operations during PC Tool sessions. Target Programming Functions cannot occur until the location of the Log File has been established. Attempting to program a controller, without first setting up the log file, will result in the display of a log error display panel indicating the Log File is not found. The User selects the "BROWSE" button to browse for the location of an existing log file. The User selects the "CREATE" button to create a Log File, and then select the "BROWSE" button to browse to the directory where the log file was created. On startup, the Settings Display Panel automatically points to the default log file and lists the path to the file in the text field adjacent to the "LOG FILE NAME" caption.

BROWSE:

The "BROWSE" Button is used to browse for the location of an existing log file. When the "BROWSE" button is pressed, an Open Dialog is displayed where the User can navigate to the directory where the Log File is stored.

CREATE:

The "CREATE" button is used to Create a Log File. Controllers cannot be programmed until a Log File has been selected. Pressing the "CREATE" button automatically builds a log file name string with the current date and time embedded in the name. A "SAVE AS" dialog is opened with the created string entered as the proposed File Name. The User may use the default created File Name or may manually enter an alternate File Name. The User is required to navigate to the directory where the Log File is to be stored. Once selected, the Log File complete path is written to the data_dump.ini file. Note: The PC Tool is initialized with a default log file setup in the directory where the executable is launched.

DONE:

The "DONE" button is used to end the Settings option selections. Press the "DONE" button when selections are complete and return to the previous panel.

Write Target

The "WRITE TARGET" button is used to Write the Local EEPROM Buffer contents to the Target EEPROM. The WRITE Target button appears, when Lab Technician Mode is enabled, after Target EEPROM has been read. Once Target EEPROM has been read the Local EEPROM Buffer contains an exact copy of Target EEPROM. The User may modify the Local EEPROM Buffer by selecting the "EDIT PROFILE" button on the Main Panel. The Edit Profile options provide the ability to change the Local EEPROM Buffer contents. The changes made during the Edit Profile Session can be written to EEPROM by selecting the WRITE TARGET button. If no changes are made to EEPROM during an Edit Profile Session, there is no point in writing to the target since the local buffer and EEPROM contents would already be the same.

SAVE PROFILE

The "SAVE PROFILE" button is used to save a copy of the Local EEPROM Buffer as read from Target EEPROM. The "SAVE PROFILE" button is displayed when the Lab Technician Mode is enabled, after Target EEPROM has been read. When the "SAVE PROFILE" button is "clicked", a "Save As" dialog is displayed. The User enters a Profile File (*.dat) name and the Local EEPROM Buffer contents is written to the User selected Profile File Name. This "SAVE PROFILE" button provides a method for capturing settings from a selected controller by writing the contents of that controller to a Profile file. This is the means for sharing settings values across a number of controllers.

EDIT PROFILE

The "EDIT PROFILE" button is displayed when Lab Technician Mode is enabled and Target EEPROM has been read. When the "EDIT PROFILE" button is "clicked", control advances to a Panel of Profile Editing options. These options include the ability to compare two profiles, the ability to compare profile 1 with the Local EEPROM Buffer contents, and the ability to modify Profile 1, Profile 2, and Local EEPROM Buffer selected data items.

EXIT PROGRAM

The "EXIT PROGRAM" button is used to Exit the PC Tool Session. When "EXIT PROGRAM" is clicked, the PC Tool interactive session ends.

EDIT PROFILE

EDIT PROFILE Options for Profile 1, Profile 2, and the Local EEPROM Buffer. This section further describes the Top Menu Panel "EDIT PROFILE" option. Edit Profile provides options for examining and modifying Profile 1, Profile 2, and the Local EEPROM Buffer. The Edit Profile session permits the user to compare a two Profile Files and compare a Profile File with the contents of the Local EEPROM Buffer. The comparisons are limited to selected data items that pertain to the operation of the controller and come in two forms. The first form is byte wise comparisons (e.g. Profile 1 Forward Speed compared with Profile 2 Forward Speed). The second form includes bitwise values provided within a selected byte (e.g. Throttle flags byte Analog Inverted Bit for Profile 1 compared with the corresponding value in EEPROM). The comparisons are side by side listings with Profile 1 presented in Column 1 and either Profile 2 or EEPROM presented in Column 2. The display is formatted into ten rows with each row containing:

    Description: A description of the field
    Profile 1 Converted Value: Data value represented in a manner that is meaningful to the User
    Profile 1 Raw Value: Raw data byte value. Toggled between Hex and Dec representation by the User
    Left Arrow: Pressing this arrow transfers the value from the second column to the first column
    Right Arrow: Pressing this arrow transfers the value from the first column to the second column
    Profile 2/EEPROM Converted Value: Data value represented in a manner that is meaningful to the User
    Profile 2/EEPROM Raw Value: Raw data byte value. Toggled between Hex and Dec representation by the User
    WRITE button: This button appears when the EEPROM is displayed in column 2 and the contents of local EEPROM buffer is different than the contents of Target EEPROM

Changes made to either Profile 1 or Profile 2 during the Edit Profile Session change the contents of the respective profile local buffer. Changing the EEPROM contents modifies the local EEPROM buffer pending a Write operation. For every value in the local EEPROM buffer that is changed, a WRITE button appears to the right of the row where the change was made. Pressing the WRITE button writes a single value from the local EEPROM buffer to the Target EEPROM. On a given Profile Edit Page, whenever the local EEPROM contents is modified or is different from Target EEPROM, a "WRITE" button appears above the column where all the row-specific "WRITE" buttons would appear. Pressing this WRITE button would write all of the changed values in the Local EEPROM Buffer that are different from the Target EEPROM contents.

UP

The Edit Profile "UP" button appears when the number of items for display is greater than the number of rows on the panel ("10"). When the "Up" button is pressed, the Edit Profile rows of comparison data "scroll up" exposing data rows off the display screen at the top.

DOWN

The Edit Profile "DOWN" button appears when the number of items for display is greater than the number of rows on the panel ("10"). When the "Down" button is pressed, the Edit Profile rows of comparison data "scroll down" exposing data rows off the display screen at the bottom.

INCR

The Edit Profile INCR button is displayed when a raw data value selection is made from one of Profile 1, Profile 2, or EEPROM. When the "INCR" button is pressed, the "selected" raw data value increments by a value of one ("1"). The "calculated value" to the left of the raw value reflects the newly changed value. The increment will continue, for each "INCR" button press until the limit for the value is reached.

DECR

The Edit Profile DECR button is displayed when a raw data value selection is made from one of Profile 1, Profile 2, or EEPROM. When the "DECR" button is pressed, the "selected" raw data value decrements by a value of one ("1"). The "calculated value" to the left of the raw value reflects the newly changed value. The decrement will continue, for each "INCR" button press until the limit for the value is reached.

Flags

Flags or Bitwise fields are bit values stored in a single byte. A byte of EEPROM data can contain up to eight bitwise fields each of which will have the value ("0") ("1"). Depending on the disposition of the individual flag signal the meaning of the value "0" can be assigned with "ON" or "OFF". On the Flags display page only the meaning of the bitwise field is display in the form of "ON" or "OFF". For those values that are eligible to be modified, the local buffer values can be toggled by clicking on the field (e.g. clicking on "OFF" changes to "ON"). For the EEPROM values, if the field is authorized to be modified in Target EEPROM, when a single bit in a data byte is changed, a single WRITE button appears. When the WRITE button is subsequently pressed, the modified byte value from the local buffer is written to Target EEPROM

[HEX/DEC]

This HEX/DEC Toggle Button option is displayed when Lab Technician Mode is enabled. Pressing this button toggles the display of the raw data values from Hex to Decimal and from Decimal to Hex format. The impact of this button is only seen on the Edit Profile Pages of specific data item values.

MISCELLANEOUS

Session Information

The operation of the PC Tool is managed by a repository for session settings called the data_dump.ini. This file resides in the directory where the executable is launched. The file manages the settings for:

    Profile Files: Stores the last Directory accessed when attaching a Profile File to the Profile Records
    Label Files: Stores the last Directory accessed when attaching a Label File to the Profile Records.
    Log File: Stores the directory and file path to the Log File where PC Tool Session information is logged
    Stores the complete list of Profile Records created from all of the interactive PC Tool Sessions.

Comm Port File

The com_port.ini file contains the initial setting value for the COMM Port connection. This file is updated from the SETTINGS Panel to reflect the selected Comm Port.
